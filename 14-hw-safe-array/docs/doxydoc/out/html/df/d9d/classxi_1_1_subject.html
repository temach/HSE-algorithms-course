Подробное описание

Класс, определяющий некоторую учебную дисциплину.

Большинство полей класса представлены в открытой области, т.к. у нас нет необходимости в их инкапсуляции в силу простоты задачи. В этом смысле класс подобен обычным структурам.

Вопрос для размышления: как ввести такое же определение «предмета» с использованием ключевого слова struct?

Документация по друзьям класса и функциям, относящимся к классу

friend

Дружественная оператор-функция вывода объекта в поток вывода.

Конструкция расширяет семантику объекта типа Subject, позволяя его использовать в потоковых операциях, например со стандартным потоком вывода:

Subject s...
cout << s;
Аргументы
[in,out]strПоток вывода.
[in]subjКонстантный объект-предмет, для вывода его текстового представления.
Возвращает
Функция возвращает тот же поток str, чтобы ее можно было использовать ... подумать, зачем это нужно.

§ description

SafeArray<std::string> xi::Subject::description = SafeArray<std::string>(MAX_LINES)

Поле описания дисциплины.

Создается в стеке (локально), несмотря на использования оператора =. Признак локальности объекта: нет ключевого слова new.

§ MAX_LINES

const int xi::Subject::MAX_LINES = 10
static

Максимальное число строк в описании дисциплины.

Ключевое слово const в действительности заставляет комлилятор выделить место для размещения неизменяемого объекта (избежать этого можно с использованием «хака» с enum. Для того, чтобы не выделять под константу место в каждом экземпляре, мы помечаем его static -ом.
